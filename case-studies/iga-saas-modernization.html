IGA Solution Modernization to SaaS

Migrating from on-premises IdentityIQ to SailPoint Identity Security Cloud for enhanced scalability and operational efficiency

Client

Financial Services

Location

USA

Migration

IIQ to ISC

Phases

5-Phase Approach

Overview

A leading financial services organization in the USA embarked on a strategic initiative to modernize its Identity Governance and Administration (IGA) infrastructure. The goal was to migrate from an on-premises SailPoint IdentityIQ (IIQ) deployment to the cloud-based SailPoint Identity Security Cloud (ISC) SaaS platform.

This transformation aimed to improve scalability, reduce operational overhead, enhance compliance capabilities, and leverage the latest identity security features available in the SaaS model.

Migration Highlights

Full feature migration from IIQ to ISC

Reverse-engineered customizations

5-phase structured migration approach

JML processes and birthright provisioning

Challenges

Complex Multi-Phase Migration

Migration involved multiple phases with intricate configurations, requiring careful planning and execution to minimize disruption.

Customization Reverse Engineering

Existing IIQ customizations required thorough reverse engineering and accurate mapping to ISC's different architecture and capabilities.

Integration & JML Processes

Integrating birthright provisioning, application connectivity, and Joiner-Mover-Leaver (JML) processes while maintaining compliance requirements.

Our Solution

5-Phase Migration Approach

1

Phase 1: Base Configuration

Configured ISC tenant based on customer requirements, established initial architecture, and set up foundational elements including authentication, authorization, and governance frameworks.

2

Phase 2: Customization Documentation

Thoroughly documented all existing IIQ customizations, reverse-engineered custom workflows, rules, and integrations to ensure accurate migration mapping.

3

Phase 3: Connectivity & Integration

Established application connectivity, integrated birthright provisioning mechanisms, and configured JML (Joiner-Mover-Leaver) processes for automated lifecycle management.

4

Phase 4: Iterative App Onboarding

Executed iterative application onboarding process, migrating applications in controlled batches with thorough testing and validation at each stage.

5

Phase 5: Validation & Cutover

Comprehensive validation testing, user acceptance testing (UAT), training, and final cutover from IIQ to ISC with minimal disruption to business operations.
